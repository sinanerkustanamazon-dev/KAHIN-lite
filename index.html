AWS Service Stack

Built for AWS Activate.

Every architectural decision optimized for the AWS ecosystem — from Free Tier compute to global edge distribution.

🖥️
Compute & Ingestion Layer

Automated cron-scheduled ingestion runs twice daily on a Free Tier instance — preventing continuous CPU burn while keeping macro data current. Sanity validation gates run inline before any data reaches scoring models.

  • Amazon EC2 t3.micro — 750hrs/month Free Tier
  • Automated cron ingestion scheduling
  • Python standard library — zero heavy dependencies
  • AWS Lambda for non-peak auto-shutdown
📦
Storage & Distribution Layer

Analytical conclusions serialized into highly compressed static JSON payloads and distributed globally via CloudFront — eliminating costly database read operations and reducing per-user infrastructure overhead to near-zero.

  • Amazon S3 — Static immutable payload store
  • Amazon CloudFront — Global CDN distribution
  • Binary compression targeting <250KB/sync
  • Cache-first architecture — minimal S3 request cost
🗄️
Data Intelligence Layer

ClickHouse handles columnar time-series storage for multi-variate financial metrics without runtime bottlenecks. PostgreSQL manages relational metadata, audit ledger hashes, and basket definitions. RDS avoided entirely for MVP phase.

  • ClickHouse — Columnar time-series engine
  • PostgreSQL — Metadata & ledger relational store
  • Redis — Caching layer for hot indicator access
  • No RDS — client-side Room DB offloads query cost

Non-Functional Requirements

Built for Scale. Built for Trust.

01
Local-First Resilience

If a sync cycle fails, the client automatically loads the last valid Room database cache. A SNAPSHOT FALLBACK MODE banner displays at the footer. The UI remains fully responsive — zero white screens for retail users.

02
Zero Data Exhaust

User watchlists, portfolio weights, and financial inputs are strictly bound to the device's native SQLite layer. No user-owned financial data is ever uploaded to the central server — eliminating data breach liability and per-user cloud database costs entirely.

03
Edge Payload Discipline

All remote data packets are binary-compressed before transmission, targeting under 250KB per daily sync cycle. On-device SQLite tables tracking user watchlists are encrypted via Android hardware storage verification contexts.

04
Data Sanity Gates

Before any sync updates local tables, incoming payloads pass client-side structural validation: payload missing ratio under 30%, close-jump outlier protection at 50% single-day threshold, and freshness window enforcement — mirroring server-side M02 quality standards.
