Product Vision
Core Mission & Strategic Objectives
Why KAHIN Lite exists and what problem it solves.
Core Mission Statement
KAHIN Lite is an uncompromisingly zero-dependency, local-first Android mobile application designed to democratize institutional-grade quantitative finance. It abstracts highly complex multi-layered macroeconomic indicators, systemic liquidity regimes, thematic supply-chain bottlenecks, and algorithmic portfolio optimizations into an intuitive, frictionless "Traffic Light" and "Single Score" consumer-facing terminal.

The application strips away the cognitive overload of institutional terminal environments, translating heavy data streams into direct risk-management guardrails for the everyday retail user. Where traditional financial platforms assume deep expertise in econometric time-series or microstructural datasets, KAHIN Lite assumes nothing — and delivers everything.

The system operates on a fundamental design principle: Symmetry of Information. The same GIRM stress signals, dark pool accumulation thresholds, and option gamma wall states that inform institutional desks at major financial institutions should be accessible to a retail user with a single tap on their Android device.

Engineering Philosophy
Four Core Engineering Principles
Non-negotiable constraints that govern every architectural decision.
  • P-1
    Symmetry of Information Bring institutional risk modeling down to a single-tap mobile interface. No financial literacy should be required to interpret a GREEN / YELLOW / RED signal or a 0–100 quality score.
  • P-2
    Leak-Proof Time Semantics The mobile client maps historical snapshots using explicit event-time boundaries (event_time, available_at), ensuring that no forward-looking data leak or look-ahead bias can infect user backtests or portfolio trackers. The invariant available_at_cutoff ≤ decision_time is cryptographically enforced at every pipeline stage.
  • P-3
    Zero-Dependency Core The client app strictly avoids heavy external SDKs, third-party tracking frameworks, or bloated charting analytics packages. This preserves extreme performance, guarantees privacy, and enables operation in resource-constrained environments without feature degradation.
  • P-4
    Local-First Resilience Total architectural insulation against cellular network dropouts, DNS routing failure, or central server downtime via a persistent local data survival framework. If the cloud is unreachable, the app remains fully operational via Room/SQLite snapshot cache.
03
System Architecture
Asymmetric Cloud-to-Edge Topology
Heavy computations in cloud. Absolute privacy on device. Zero per-user database cost.

To scale to millions of active retail units without incurring prohibitive cloud computing bills, KAHIN Lite rejects the traditional microservices layout in favor of an asymmetric architecture: heavy computations concentrated in the cloud, absolute privacy executed on the edge.

Pipeline Flow Diagram
# External Data Sources (Public APIs) [FRED API] · [Yahoo Finance] · [SEC Filings] · [Exchange Feeds] │ ▼ ← Executed Once Daily via Automated Cron [Central Cloud Compute Engine — AWS EC2] Runs GIRM Stress Lab, Score Engines & Audit Ledgers │ ▼ ← Serialized into Highly Compressed Static JSON [Cloud API Edge Layer — AWS S3 + CloudFront] Lightweight, Stateless CDN & Proxy Distribution Nodes │ ▼ ← Single-Burst Compressed Synchronization over TLS [Android Client] └─> Local Database (Room/SQLite Cache) └─> On-Device Sanity Validation Gates └─> Isolated UI Rendering Layer
SECTION 2.1 — LOCAL DATA ISOLATION & CLIENT PRIVACY MODE
Zero Data Exhaust Guarantee
User watchlists, personal portfolio asset balances, arithmetically input values, and search history trajectories are strictly bound to the client device's native SQLite database layer. No user-owned financial data or telemetry is ever uploaded to the central server cluster. This eliminates data breach vulnerabilities and slashes cloud database infrastructure costs to zero per user.
SECTION 2.2 — CLIENT-SIDE DATA SANITY GATES

Before the local database updates its underlying tables from an incoming API sync burst, it subjects the incoming payload to local structural sanity evaluations — mirroring the server-side M02 quality pipeline:

  • Gate 1
    Payload Missing Ratio If more than 30% of expected data points return missing parameters, the sync batch is rejected entirely and the client forces a fallback to the last valid historical state.
  • Gate 2
    Close Jump Outlier Protection If an asset's price or macro score exhibits an uncorroborated single-day jump exceeding 50%, the client flags an anomaly warning badge on the UI and logs the record under investigation.

Functional Requirements · FR-1
Adaptive Macro Weather Dashboard
Integrates M11 Composer authority routing with M04 GIRM stress scoring.

Upon application launch, the networking module executes a handshake with the edge configuration gateway to process the prevailing global macro systemic risk state calculated by KAHIN's authority routing hierarchy — a strict three-tier override system that governs the entire app state.

AUTHORITY ROUTING HIERARCHY
  • Tier 1 — GIRM
    Highest Authority — Emergency Override If Financing, Demand, Geopolitical, or AI Capex stress scores cross institutional limits (GIRM ≥ 4.0), the app overrides all local settings and triggers a hard system alert regardless of any other indicator state.
  • Tier 2 — Sentinel
    Second Authority — Conservative Override Restricts all technical trend tools and forces conservative warning states if liquidity thresholds drop (Sentinel ≤ 45).
  • Tier 3 — Regime
    Third Authority — Watchlist Mode Places the terminal into mandatory watchlist mode if macro contradictions expand (Regime < 45), blocking new position signals.
UI STATE CONFIGURATIONS
GREEN — Constructive / Neutral Regime
Liquidity expanding, low systemic friction. Interface utilizes deep slate/charcoal tones with sharp jade accents. All modules unrestricted. Capital deployment signals: CONSTRUCTIVE or NEUTRAL.
YELLOW — Risk-Reduced / Watchlist-Only Regime
Liquidity deceleration or macro contradiction flagged. Interface displays amber alert indicators warning users to optimize cash positions. Speculative theme access visually restricted.
RED — GIRM Emergency Brake Engaged
Systemic financial or geopolitical stress crosses critical limits. Entire viewport adopts high-contrast crimson theme. Defensive allocation model forces 65% safe haven, 25% Iron Dome, 10% max speculative.
Vernacular Masking Protocol
The system intercepts complex structural indicators — such as TED spreads, Money Market Fund dynamics, or Reverse Repo drains — and maps them into clear, actionable, non-academic plain-language panels. No financial jargon reaches the retail user.

Explainable Audit Drivers
The UI surfaces precisely three human-readable bullet items indicating why the asset received its score, extracted directly from the underlying evaluation engine's top-4 scoring AuditDriver factors. The user always knows the "why" behind the number.

Asymmetric Cloud-to-Edge Design Rationale
By serving analytical conclusions as static JSON via CloudFront CDN rather than dynamic database queries, KAHIN Lite scales to millions of active users at a flat per-user cloud cost of zero. The only variable cost is the once-daily EC2 ingestion cycle — covered by Free Tier.

Budget Management Best Practices
1. Budget Alerts: AWS Budgets configured to trigger email at $8.00/month.
2. Resource Lifecycle: Monthly audit of unused volumes and snapshots to prevent silent billing creep.
3. Database Strategy: RDS avoided entirely for MVP. Client-side Room DB offloads all query cost.
4. Auto-Shutdown: Lambda functions stop EC2 outside peak ingestion windows automatically.
